Entrypoint (docker/entrypoint.sh)

Auto-migration no deploy: O entrypoint.sh roda php artisan migrate --force automaticamente quando o container app inicia. Novas tabelas (ex: pipeline_user) são criadas automaticamente em cada deploy sem intervenção manual.

O entrypoint executa os seguintes passos na inicialização do container app:

EtapaDescrição
chown storage/Garante permissões de escrita para www-data
Sync public/Copia assets para volume compartilhado com nginx
Wait MySQLAguarda MySQL estar pronto (retry a cada 3s)
Wait RedisAguarda Redis estar pronto (retry a cada 2s)
package:discoverRedescobre pacotes (limpa cache antigo antes)
migrate --forceRoda todas as migrations pendentes automaticamente
db:seedExecuta seeders apenas se nenhum usuário existe no banco
Cacheconfig:cache, route:cache, view:cache, storage:link
Migrations com erro: Se uma migration falhar (ex: tabela já existe em re-deploy), o entrypoint loga o warning e continua a inicialização normalmente. O container não é interrompido.

Nginx Config

# Rotas normais → PHP-FPM
location / {
    try_files $uri $uri/ /index.php?$query_string;
}

# WebSocket → Reverb
location /app/ {
    proxy_pass http://reverb:8080;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

location /apps/ {
    proxy_pass http://reverb:8080;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

Reverb (WebSocket)

Path no config PHP: reverb.apps.apps.0.key (dois .apps). Cuidado ao acessar programaticamente.
  • Reverb escuta em 0.0.0.0:8080 (interno)
  • Browser conecta via wss://app.syncro.chat/app/{key}
  • Config injetada no Blade: window.reverbConfig = { key, host, port, scheme }

CI/CD

git push main │ └──> Build Docker Image │ ├── composer install --no-dev ├── npm ci && npm run build ├── php artisan config:cache ├── php artisan route:cache └── docker push matolado/crm:latest │ └──> Portainer detects new image │ └──> Docker Swarm rolling update │ └──> entrypoint.sh runs migrate --force
Tabela pipeline_user: Criada automaticamente pela migration 2026_03_11_000006_create_pipeline_user_table no próximo deploy. Não requer intervenção manual -- o entrypoint.sh executa php artisan migrate --force na inicialização.

Comandos de Debug (VPS)

# Logs do nginx
docker service logs syncro_nginx --tail 50

# Logs do WhatsApp (tempo real)
docker exec $(docker ps -q -f name=syncro_app) \
    tail -f /var/www/storage/logs/whatsapp-$(date +%Y-%m-%d).log

# Logs do Laravel
docker exec $(docker ps -q -f name=syncro_app) \
    tail -f /var/www/storage/logs/laravel.log

# Artisan dentro do container
docker exec $(docker ps -q -f name=syncro_app) php artisan migrate --force

# Redis CLI
docker exec -it $(docker ps -q -f name=syncro_redis) redis-cli

# Restart de um service
docker service update --force syncro_app

# Status dos services
docker service ls | grep syncro
