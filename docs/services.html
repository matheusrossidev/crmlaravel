Fluxos de Processamento

WhatsApp Inbound Message

WAHA ──POST──> WhatsappWebhookController │ ├── Dedup: Cache::add("waha:processing:{msgId}", 1, 10) │ └──> ProcessWahaWebhook::dispatchSync() │ ├── Encontra/cria WhatsappConversation ├── Salva WhatsappMessage (UNIQUE waha_message_id) ├── Encontra/cria Lead se pipeline.auto_create_from_whatsapp ├── Broadcast: WhatsappMessageCreated + WhatsappConversationUpdated ├── AutomationEngine::run('message_received') ├── LeadScoringService::evaluate('message_received') │ ├── Se chatbot_flow_id ativo: │ └──> ProcessChatbotStep::dispatchSync() │ └── Se ai_agent_id ativo: └──> ProcessAiResponse::dispatch() (queued, com debounce)

Website Widget Message

Widget JS ──POST──> WebsiteWidgetController::message() │ └──> WebsiteChatService::processMessage() │ ├── resolveInput() — processa resposta do visitor ├── executeFromCursor() — executa steps do fluxo │ ├── message → adiciona ao replies[] │ ├── input → para e pede resposta │ ├── condition → avalia e segue branch │ └── action → create_lead, change_stage, etc. ├── persistState() — salva cursor + vars no DB └── return {replies, buttons, input_type}

AI Agent Response

ProcessAiResponse (queued job) │ ├── Debounce: verifica versao (ignora se outdated) ├── checkTokenQuota() — soma base + incrementos do mes │ ├── Se agent.use_agno: │ └──> AgnoService::chat() — resposta + acoes │ └── Senao: chamada direta ao LLM (OpenAI/Anthropic) │ ├── Aplica acoes: setStage, addTags, notifyIntent, transfer, calendar ├── splitIntoMessages() — divide resposta longa ├── sendWhatsappReplies() — envia sequencialmente └── Log token usage em AiUsageLog

Nurture Sequence Flow

sequences:process (a cada minuto) │ ├── Busca LeadSequences com status=active e next_step_at <= now() │ └──> NurtureSequenceService::processStep() │ ├── Verifica exit conditions (lead respondeu? humano enviou msg?) ├── Verifica IA/Chatbot ativo → pausa se sim ├── Verifica opted_out → exit se sim │ ├── Executa step: │ ├── message → cria ScheduledMessage │ ├── action → move_stage, add_tag, assign_user │ ├── wait_reply → aguarda timeout │ └── condition → avalia e pode pular para step_if_false │ └── advanceToNextStep() ou markCompleted()

Sales Goal Lifecycle

goals:process-recurrence (diario 00:05) │ ├── Busca goals expirados (end_date < today) │ ├── generateSnapshot() — salva historico │ └── Se is_recurring → renewRecurring() (aplica growth_rate) │ goals:check-alerts (diario 09:00) │ ├── Calcula progress() e forecast() para cada goal ativo └── Se pace=behind ou acceleration_needed > 50% → GoalAlertNotification
