Padrões Comuns nos Models

Timestamps

Vários models usam $timestamps = false e gerenciam created_at manualmente:

  • LeadEvent — sempre passar 'created_at' => now() no ::create()
  • PipelineStage, Sale, LostSale, LostSaleReason
  • WhatsappConversation, WebsiteConversation, WebhookLog
  • UpsellTriggerLog — timestamps=false, usar fired_at

Casts Comuns

  • 'tags' => 'array' — Leads e Conversations armazenam tags como JSON
  • 'chatbot_variables' => 'array' — Estado do chatbot em conversations
  • 'features_json' => 'array', 'config' => 'array' — Config flexível em JSON
  • 'value' => 'decimal:2' — Valores monetários em leads e sales
  • 'dashboard_config' => 'array' — Configuração de dashboard do User

User

Sem BelongsToTenant — O model User não usa o trait BelongsToTenant. O escopo de tenant é feito via tenant_id no registro, mas sem Global Scope automático.

Controla quais pipelines cada usuário pode acessar.

ColunaTipoDescrição

Retrocompatível — Se um usuário não tem nenhuma entrada na tabela pipeline_user, ele vê todas as pipelines (comportamento padrão). Entradas são criadas apenas quando se deseja restringir o acesso. Admins (isAdmin() = true) sempre veem todas, independente da pivot.
